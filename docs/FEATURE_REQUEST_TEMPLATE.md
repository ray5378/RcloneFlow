# FEATURE_REQUEST_TEMPLATE.md

功能需求模板。

用于帮助开发者更高效地提需求，减少“想法有了，但边界不清、改动范围发散”的问题。

---

## 推荐模板

### 1. 背景
- 这是哪个页面 / 哪个功能 / 哪个使用场景

### 2. 当前问题
- 现在不好用的地方是什么
- 哪里的体验最差

### 3. 预期目标
- 希望最终达到什么效果

### 4. 限制条件
- 这次不要动什么
- 哪些地方这轮不希望扩展

### 5. 优先级
- 为什么这件事现在值得做
- 是否比其他问题更优先

---

## 一个简单原则

提需求时，把“背景、问题、预期、限制”说清楚，通常比一开始就给解决方案更重要。
