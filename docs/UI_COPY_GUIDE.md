# UI_COPY_GUIDE.md

界面文案指南。

这份文档用于约束前端展示文案的基本风格和方向，减少界面里出现语义模糊、用户误解或前后不一致的表达。

---

## 1. 总目标

界面文案应优先做到：
- 清楚
- 一致
- 不误导
- 能帮助用户理解当前状态

不要为了“看起来专业”而牺牲可理解性。

---

## 2. 关键原则

### 2.1 不同语义要用不同文案
例如：
- 预估值 -> 应明确体现“预估”
- 实时值 -> 应明确体现“当前/实时”
- 完成态 -> 应体现“已完成/最终”

不要把不同语义用同一种词糊在一起。

### 2.2 不要把不确定值说成确定值
如果一个值来自回退、估算或兼容逻辑，文案上应避免让用户误以为它是绝对准确的实时结果。

### 2.3 错误和提示要可理解
错误文案应尽量告诉用户：
- 出了什么问题
- 当前会造成什么影响
- 下一步更可能怎么做

---

## 3. 当前项目里尤其要注意的文案点

- `预估总量` 与 `当前总量` 应避免混淆
- 运行中 ETA 不应给出看似精确、实际不可信的结果
- 调试信息和普通用户信息要明显区分
- 运行中卡片、小窗、历史记录的相近概念应尽量统一用词

---

## 4. 一个简单原则

如果一个用户看到文案后，更难判断“现在是什么状态”，那这条文案通常还不够好。
